## Key Takeaways

- **EC2 Instance Types**: Different families are optimized for specific workloads. General purpose (M5), compute optimized (C5), memory optimized (R5), and burstable performance (T3) instances each serve different performance and cost requirements.

- **Purchasing Options**: Understanding the trade-offs between flexibility and cost savings is critical. On-Demand offers maximum flexibility, while Reserved Instances and Savings Plans provide significant discounts for committed usage. Spot Instances offer the lowest cost but can be interrupted.

- **Amazon Machine Images (AMI)**: AMIs serve as templates that allow consistent and repeatable instance launches. Creating custom AMIs helps reduce configuration drift and speeds up deployment.

- **EC2 Image Builder**: This service automates the process of building, testing, and maintaining custom AMIs. It reduces manual effort and helps minimize human error when creating standardized server images.
