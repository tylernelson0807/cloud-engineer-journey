## Key Takeaways

- Successfully completed the role-play on Other Compute Services with improved clarity and reduced use of technical jargon, following previous feedback. The ability to explain Lambda, Lightsail, ECS, and EKS in simple, business-relevant terms is strengthening.

- AWS Lambda provides serverless compute for event-driven workloads with minimal operational overhead. Amazon Lightsail offers a simplified virtual server experience with predictable pricing. Amazon ECS and EKS provide container orchestration options, with Fargate enabling a serverless container experience.

- AWS CloudFormation enables Infrastructure as Code by allowing infrastructure to be defined in JSON or YAML templates. This supports consistent, repeatable, and version-controlled deployments across environments.

- AWS CDK allows infrastructure to be defined using familiar programming languages (such as TypeScript or Python), which are then converted into CloudFormation templates. This approach improves developer productivity while maintaining the benefits of CloudFormation.

- Deployment strategies such as Blue/Green and Canary deployments, along with tools like Elastic Beanstalk and Systems Manager, help reduce risk and operational effort when releasing changes at scale.
